# ERC8004 — Immediate Collaboration Requests (D0)

Obiettivo: allineare R&D e business per chiudere 1 pilot entro 14 giorni.

## Richieste immediate (priorità alta)
1. **Architecture brief (48h)**
   - Stack minimo raccomandato per 1–2 agenti in produzione leggera
   - Vincoli tecnici, costi infra stimati, dipendenze critiche
2. **Use-case feasibility matrix (72h)**
   - Valutazione fattibilità (alto/medio/basso) per:
     - Lead qualification agent
     - Customer support triage agent
     - Document intake/QA agent
   - Per ogni use case: effort, rischio, dati richiesti, tempi di setup
3. **Guardrails & compliance baseline (72h)**
   - Checklist sicurezza minima (accessi, logging, fallback umano)
   - Policy consigliate per clienti EU (privacy, data retention)
4. **Pilot technical blueprint (entro D5)**
   - Template implementazione riusabile per pilot 14 giorni
   - KPI tecnici consigliati: latency, error rate, handoff rate, uptime
5. **Demo assets (entro D6)**
   - 2 demo flow pronti per sales call
   - Script di validazione tecnica live (5–10 min)

## Decisioni congiunte richieste (Business + ERC8004)
- Quale use case ha miglior trade-off: valore commerciale vs complessità tecnica?
- Limiti dichiarabili in offerta per evitare overpromise.
- Soglia minima KPI per passare da pilot a retainer.

## Cadence proposta
- Sync breve giornaliero: 15 min
- Review congiunta: D3, D7, D10, D14
- Single source of truth condivisa: backlog requisiti + assunzioni validate
